# План реализации ChatList

Поэтапный порядок действий по созданию программы на основании `PROJECT.md`.

---

## Этап 0. Подготовка проекта

1. Обновить `requirements.txt`: PyQt6, httpx (или requests), python-dotenv.
2. Добавить `.env.example` с шаблонами имён переменных для API-ключей (без реальных секретов).
3. Убедиться, что `.env` указан в `.gitignore`.
4. Создать структуру каталогов и заготовки модулей:
   - `db.py` — работа с SQLite
   - `models.py` — логика моделей нейросетей
   - `network.py` — HTTP-запросы к API
   - `main.py` — GUI

---

## Этап 1. База данных (`db.py`)

1. Реализовать подключение к SQLite и создание файла БД при первом запуске.
2. Создать таблицы по схеме из `DATABASE.md`:
   - `prompts`
   - `models`
   - `results`
   - `settings`
3. Написать CRUD-функции:
   - промты: добавить, список, поиск по тексту/тегам;
   - модели: добавить/редактировать, список активных (`is_active = 1`);
   - результаты: сохранить выбранные строки, список сохранённых;
   - настройки: get/set по ключу.
4. Проверить создание схемы и базовые операции без GUI.

---

## Этап 2. Сеть и модели (`network.py`, `models.py`)

1. В `models.py` описать работу со списком нейросетей из БД (активные модели, чтение `api_key_env`).
2. В `network.py` реализовать отправку промта по `api_url` с ключом из `.env`.
3. Сделать единый интерфейс ответа: модель → текст ответа / ошибка.
4. Поддержать параллельную (или последовательную) отправку в несколько моделей.
5. Проверить запросы на одной тестовой модели.

---

## Этап 3. Временная таблица результатов (в памяти)

1. Описать структуру строки результата: модель, текст ответа, `selected`.
2. Реализовать создание временной таблицы после ответов API.
3. Реализовать очистку при новом запросе и после сохранения в БД.

---

## Этап 4. GUI — каркас (`main.py`)

1. Главное окно: зона ввода промта, список/выбор сохранённых промтов, кнопка «Отправить».
2. Таблица результатов: модель, ответ, чекбокс `selected`.
3. Кнопка «Сохранить» для записи выбранных строк в `results`.
4. Базовые экраны/диалоги для управления моделями и просмотра сохранённых результатов.

---

## Этап 5. Рабочий процесс end-to-end

1. Связать UI с `db.py`, `models.py`, `network.py`.
2. Реализовать сценарий из спецификации:
   - ввод или выбор промта;
   - отправка в активные модели;
   - показ временной таблицы;
   - сохранение отмеченных строк;
   - очистка временной таблицы при новом запросе.
3. Сохранять новый промт в `prompts` (дата, текст, теги — по возможности на этом этапе).
4. Прогнать полный сценарий вручную.

---

## Этап 6. Настройки и конфигурация

1. Экран/диалог настроек на базе таблицы `settings`.
2. Загрузка API-ключей только через `.env` (в БД — имена переменных).
3. Проверка корректной работы при отсутствии ключа (понятное сообщение об ошибке).

---

## Этап 7. Доработки (опционально)

1. Поиск и сортировка во всех таблицах UI.
2. Экспорт выбранных результатов в Markdown / JSON.
3. Адаптеры под разные типы API (OpenAI, DeepSeek, Groq и т.д.).
4. Логирование запросов.

---

## Этап 8. Сборка и документация

1. Обновить `README.md`: установка, `.env`, запуск.
2. Собрать исполняемый файл (PyInstaller), если потребуется.
3. Финальная проверка сценария на собранной версии.
